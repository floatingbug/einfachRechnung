### Offer → Invoice Flow

- User öffnet Angebot
- User wählt "In Rechnung umwandeln"
- Angebotsdaten werden übernommen
- Kundendaten werden übernommen
- Positionen werden übernommen
- Rechnungssummen werden neu berechnet
- Rechnungsnummer wird erzeugt
- Rechnungsdatum wird gesetzt
- Zahlungsziel wird gesetzt
- Rechnung speichern
- Rechnung erhält Status "draft"
- Angebot erhält Status "converted"
- User kann Rechnung senden
- Nach Versand erhält Rechnung Status "sent"


### Create Offer

- User startet "Neues Angebot"
- User wählt Kunde aus
- Falls Kunde nicht existiert → Kunde erstellen
- User erfasst Angebotspositionen
- Angebotssumme berechnen
- Angebotsdatum setzen
- Gültigkeitsdatum setzen
- Angebot speichern
- Angebot erhält Status "draft"
- User kann Angebot senden
- Nach Versand erhält Angebot Status "sent"
